# Built-in Nodes Types


Unigine provides a set of built-in node types allowing you to create a world containing practically all of the objects present in real life. For convenience, they are formed into several groups that are responsible for different kinds of operations:


- **[Nodes](../objects/nodes/index.md)** include fundamental objects that form the world (nodes and node references), dummy nodes, layers, pivots and triggers.
- **[Objects](../objects/objects/index.md)** represent imitations of entities present in the real world: objects, sky, terrains, water, etc.
- **[Effects](../objects/effects/index.md)** contain particles systems, physical fields, volumetric objects, etc.
- **[Light sources](../objects/lights/index.md)** include objects providing different kinds of the scene illumination: global, omni-directional, projected, etc.
- **[Decals](../objects/decals/index.md)** contains decals, a material projection on a surface (perfect for bloodstains, ashes, roads, etc).
- **[Geodetics](../objects/geodetics/index.md)** include objects providing geodesy math.
- **[World Objects](../objects/worlds/index.md)** represent invisible nodes used for world management.
- **[Sound Objects](../objects/sounds/index.md)** represent sound sources and reverberations.
- **[Pathfinding Objects](../objects/navigations/navigation/index.md)** represent objects used for pathfinding.
- **[Players](../objects/players/index.md)** represent different kinds of cameras.


> **Notice:** Note that in terms of Unigine all of the objects, added into the scene (regardless of their extension), are displayed in the Nodes editor and called nodes.
