# Cloud Deployment and Web Streaming


Delivering complex, interactive 3D applications today can follow several deployment paths — from traditional desktop installations to fully cloud-hosted streaming. Each approach offers a different combination of performance, accessibility, and data security benefits. Here is the comparison between different deployment options for a typical complex project with interactive 3D visualization:


[![](web_streaming_comparison.png)](web_streaming_comparison.png)


UNIGINE offers a dedicated *[WebStream](../code/plugins/webstream/index.md)* Plugin that enables you to take advantage of using web streaming for interactive 3D applications. Here are some of the key benefits:


**1. *Instant High-End 3D on Any Device*:** Render high-fidelity 3D scenes in any browser - no installs or powerful hardware needed.


**Use Cases:** *Product Configurators, Architectural Previews, Trade Shows, Investor Presentations, Marketing Demos, Digital Twins, Virtual Prototypes*


- Server-side GPU rendering
- 2-way interaction on tablets, phones, or low-spec laptops
- Same output for every user - no hardware variation or quality compromise
- Ideal for demos, reviews, and field access


**2. *Protection of Your 3D Assets*:** Only pixels are streamed - assets never leave the server - no extraction risk unlike web or desktop apps.


**Use Cases:** *Proprietary Models, Confidential Simulations, IP-Sensitive Projects*


- No downloads or client caching
- Meets enterprise and NDA-level security


**3. *Streaming of Massive 3D Environments Without Long Downloads*:** instant interaction with huge 3D environments - no heavy data overhead of web apps and long setup time of desktop apps.


**Use Cases:** *City-scale Digital Twins, Engineering Projects, or GIS-based Applications*


- No pre-loading of gigabytes of 3D data
- Immediate access from a browser
- On-demand visualizations for large-scale content


**4. Centralized Updates, Instant Distribution:** consistency across desktop or web deployments for regularly-changed apps (bug fixes, new features, asset updates).


**Use Cases:** *B2B Platforms, Training Simulations, or Support-Heavy Deployments*


- Update once on the server - all users get it instantly
- No client-side dependencies
- Reduced support overhead and version drift


**5. Delivery of Time-Limited or Metered 3D Sessions:** session time control, concurrent users limits, or integrated billing. Great for commercial, training, or license-limited 3D apps.


**Use Cases:** *Training Portals, Subscription-Based Simulations, or Guided Demos*


- On-demand 3D sessions
- Usage tracking, access rules enforcement
- Paid access or restricted content use
